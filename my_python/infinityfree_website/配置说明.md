# ⚙️ InfinityFree 异步消息系统 - 配置说明

## 🔴 重要提示

**出现 "Connection timed out" 错误的原因：**

```
❌ 发送失败：Connection timed out
```

这是因为程序尝试连接到 **尚未部署或未正确配置** 的 InfinityFree 网站。

---

## ✅ 正确的使用流程

### 第一阶段：部署网站（必须完成）

#### 1. 注册 InfinityFree
- 访问 https://infinityfree.net
- 注册账号
- 创建免费网站
- 获得域名：`yourname.infinityfreeapp.com`

#### 2. 上传文件到 htdocs
**必须上传以下文件到 `/htdocs` 目录：**
- config.php
- database.sql
- receive.php
- check_status.php
- get_pending_messages.php
- submit_response.php
- index.html
- check_config.php

#### 3. 配置数据库
编辑 `config.php`，填入从 InfinityFree 控制面板获取的信息：
```php
define('DB_HOST', 'sql123.infinityfree.com');  // 你的数据库主机
define('DB_USER', 'if0_12345678');             // 你的数据库用户
define('DB_PASS', 'your_password');            // 你的密码
define('DB_NAME', 'epiz_12345678_mydb');       // 你的数据库名
```

#### 4. 创建数据表
- 打开 phpMyAdmin（在 InfinityFree 控制面板）
- 选择你的数据库
- 执行 `database.sql` 中的 SQL 语句

#### 5. 验证部署
访问：`https://yourname.infinityfreeapp.com/check_config.php`

确保所有检查都显示 ✅

---

### 第二阶段：配置电脑端处理器

#### 修改 BASE_URL

编辑文件：`ptools/my_llm/async_processor.py`

**第 19-21 行，修改为：**
```python
# ⚠️ 重要：必须修改为你的实际网站地址
# 示例：BASE_URL = "https://yourname.infinityfreeapp.com"
BASE_URL = os.getenv("INFINITYFREE_URL", "https://your-domain.infinityfreeapp.com")
```

**两种配置方法：**

##### 方法 1：直接修改代码（推荐）
```python
BASE_URL = "https://yourname.infinityfreeapp.com"  # 替换为你的真实域名
```

##### 方法 2：使用环境变量
```bash
# Windows PowerShell
$env:INFINITYFREE_URL="https://yourname.infinityfreeapp.com"
python ptools\my_llm\async_processor.py

# Windows CMD
set INFINITYFREE_URL=https://yourname.infinityfreeapp.com
python ptools\my_llm\async_processor.py
```

## ❗ 常见错误及解决方案

### 错误 1: Connection timed out

**原因：**
- BASE_URL 未正确配置
- 网站尚未部署到 InfinityFree
- 网络无法访问

**解决：**
1. 确认已完成网站部署
2. 访问 `https://yourname.infinityfreeapp.com/check_config.php` 验证
3. 修改 `async_processor.py` 中的 BASE_URL

### 错误 2: 404 Not Found

**原因：**
- PHP 文件未正确上传
- 文件名拼写错误

**解决：**
1. 检查所有 PHP 文件是否上传到 htdocs
2. 确认文件名完全正确（区分大小写）

### 错误 3: Database connection failed

**原因：**
- config.php 配置错误
- 数据库未创建

**解决：**
1. 从 InfinityFree 控制面板复制正确的数据库信息
2. 更新 config.php
3. 在 phpMyAdmin 中执行 database.sql

### 错误 4: No pending messages

**原因：**
- 数据库中确实没有待处理消息
- 这是正常状态

**解决：**
- 这是正常的！说明所有消息都已处理
- 去网站发送新消息测试

## 💡 理解架构

### 关键点

✅ **网站只是存储器**
- InfinityFree 网站 = 云端数据库
- 24 小时接收消息
- 存储待处理和已处理的消息

✅ **电脑端是处理器**
- 主动获取待处理消息
- 本地 Ollama 处理（不需要联网调用 API）
- 处理后返回到数据库

✅ **完全异步**
- 不需要实时连接
- 可以在任何时间处理
- 用户稍后查看结果

✅ **零成本**
- InfinityFree 免费托管
- 本地 Ollama 免费使用
- 无需购买服务器或 API
